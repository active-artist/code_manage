# AGENTS.md

## 项目概览

这是一个本地静态网页工具，名称为“刷题复盘台”。项目用于管理 LeetCode 或机试刷题记录，包括题目信息、代码、题目描述、核心思路、易错点和复习状态。

目标是帮助用户在复试或机试准备中快速记录题目、筛选题库、查看代码、进行随机抽查和复盘。

## 技术栈与运行方式

- 技术栈：原生 HTML、CSS、JavaScript。
- 构建方式：无构建工具、无依赖安装步骤。
- 运行方式：直接用浏览器打开 `index.html`。
- 数据存储：浏览器 `localStorage`。

## 文件结构

```text
.
├── index.html   # 页面结构和主要 DOM 节点
├── styles.css   # 页面布局、视觉样式、响应式适配
├── app.js       # 状态、数据、筛选、复习、导入导出和交互逻辑
├── README.md    # 面向用户的项目说明
└── AGENTS.md    # 面向 Codex/开发者的项目说明
```

## 核心功能

- 个人主页：展示题目总数、今日新增、待复习、薄弱题、近一年添加热力图、状态分布、近期动态和复习建议。
- 提交页面：录入或编辑题号、题名、LeetCode 链接、难度、语言、标签、题目描述、核心思路、易错点和代码。
- 题库记录：支持关键词、分类、状态筛选；支持“部分代码 / 所有代码”切换；代码以编辑器风格展示。
- 复习页面：支持随机抽查题目、隐藏/显示代码、查看题目简介、思路提示、易错点和复习次数，并标记为“掌握 / 模糊 / 不会”。
- 数据管理：支持导出 JSON 和导入 JSON。
- 题目信息抓取：可通过 LeetCode 链接尝试调用 LeetCode CN GraphQL 获取题号、题名、难度、标签和题目描述。

## 数据与存储

题目数据保存在 `localStorage` 中，当前主要存储键为 `leetcode-review-desk-v2`，并兼容旧键 `leetcode-review-desk-v1`。

单条题目记录通常包含：

- `id`：本地唯一 ID。
- `number`：题号。
- `title`：题名。
- `url`：LeetCode 链接。
- `difficulty`：难度，取值为 `Easy`、`Medium`、`Hard`。
- `language`：代码语言，默认 `C++`。
- `tags`：分类标签数组。
- `description`：题目描述。
- `idea`：核心思路。
- `pitfall`：易错点。
- `code`：代码内容。
- `status`：复习状态，当前为“掌握 / 模糊 / 不会 / 未复习”。
- `reviewCount`：复习次数。
- `lastReviewedAt`：上次复习时间。
- `createdAt`：创建时间。
- `updatedAt`：更新时间。

## 开发维护约定

- 保持项目纯静态可用，不要随意引入 React、构建工具、包管理器或后端服务。
- 修改中文文案时注意文件编码，避免出现乱码。
- 修改数据结构时要在 `normalizeProblem()` 中兼容旧数据。
- 修改复习状态文案时，要同步检查筛选、统计、权重、导入数据归一化和 UI 展示。
- LeetCode 抓取依赖浏览器直接请求 `https://leetcode.cn/graphql/`，可能受跨域或网络策略影响；失败时必须允许用户手动填写。
- 代码展示由 JavaScript 动态生成行号和基础高亮，改动时注意避免直接插入未转义的用户代码。
- 热力图统计基于题目 `createdAt`，表示添加题目次数，不表示复习次数。

## 验证建议

- 文档变更通常不需要运行前端测试。
- 修改 `app.js` 后建议运行：

```powershell
node --check app.js
```

- 修改页面或样式后建议在浏览器中检查：
  - 桌面和移动端布局是否正常。
  - 个人主页热力图是否可横向滚动。
  - 提交、题库筛选、复习抽查、导入导出是否可用。
  - 旧的 localStorage 数据是否仍能正常显示。
