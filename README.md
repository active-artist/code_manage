# LeetCode 复试复习台

一个用于管理考研复试机试刷题代码的本地网页工具。它的目标不是单纯保存代码，而是帮助你按分类复习、随机抽查、记录掌握情况，并训练复试时的口述解题能力。

## 当前功能

- 录入 LeetCode 题目：题号、题名、链接、难度、语言、分类标签、核心思路、易错点和代码。
- 题库管理：支持添加、编辑、删除题目。
- 筛选搜索：可按关键词、分类标签、复习状态筛选。
- 随机抽查：优先抽取 `不会`、`模糊`、`未复习` 或较久没有复习的题目。
- 口述模式：抽查时先隐藏代码，先自己复述思路，再显示代码对照。
- 复习状态：可标记为 `掌握`、`模糊`、`不会`。
- 数据管理：数据保存在浏览器本地，支持导入和导出 JSON。

## 快速开始

本项目是纯静态网页，不需要安装依赖。

直接用浏览器打开：

```text
index.html
```

在 Windows 中也可以双击 `index.html` 使用。

## 项目结构

```text
.
├── index.html   # 页面结构
├── styles.css   # 页面样式
├── app.js       # 题库、筛选、抽查、导入导出等逻辑
└── README.md    # 项目说明
```

## 数据保存方式

当前版本使用浏览器的 `localStorage` 保存数据。

优点：

- 不需要后端服务。
- 不需要数据库。
- 打开网页即可使用。

注意：

- 数据和浏览器绑定，更换浏览器或清理浏览器数据可能导致本地记录丢失。
- 建议定期点击页面右上角的 `导出 JSON`，把题库备份到仓库或其他安全位置。

## 推荐使用流程

1. 每写完一道 LeetCode，把代码、核心思路和易错点录入工具。
2. 给题目打上分类标签，例如 `动态规划`、`二叉树`、`回溯`、`图论`、`贪心`、`字符串`。
3. 每天复习时先用分类筛选薄弱模块。
4. 点击 `抽查一题`，先不要看代码，尝试口述：
   - 题目考点是什么
   - 为什么选这个算法
   - 状态、递推、边界或搜索过程是什么
   - 时间复杂度和空间复杂度是多少
5. 复述后显示代码，对照自己的实现。
6. 根据结果标记 `掌握`、`模糊` 或 `不会`。

## Git 管理建议

建议把项目源码提交到 Git：

```powershell
git add index.html styles.css app.js README.md
git commit -m "Add leetcode review desk"
```

如果你导出了题库 JSON，并希望 Git 也管理复习数据，可以建立一个目录：

```text
data/
└── leetcode-review-backup.json
```

然后定期导出覆盖该文件，再提交：

```powershell
git add data/leetcode-review-backup.json
git commit -m "Update leetcode review data"
```

如果不希望题库数据进入 Git，可以把导出的 JSON 放到仓库外部，或后续添加 `.gitignore` 忽略备份文件。

## 后续可扩展功能

比较适合下一步添加的功能：

- 从代码目录批量导入 `.cpp`、`.py`、`.java` 文件。
- 支持按复习间隔生成今日待复习列表。
- 增加复杂度字段：时间复杂度、空间复杂度。
- 增加复试口述模板：思路、边界、复杂度、可优化点。
- 使用 `data/problems.json` 保存题库，减少对浏览器本地存储的依赖。
- 后续题量变多后升级为 SQLite 或本地后端服务。

## 维护原则

- 先保持工具轻量，优先服务复试复习效率。
- 新功能尽量围绕三个问题设计：
  - 能不能更快找到要复习的题？
  - 能不能更准确发现薄弱题？
  - 能不能帮助复试时讲清楚解法？
- 不为了炫技引入复杂框架，除非现有静态网页已经明显不够用。
